Sync [PersistentVolumes](https://kubernetes.io/docs/concepts/storage/persistent-volumes/) from the virtual cluster to the host cluster. By default, this is disabled and so pseudo PersistentVolumes are created instead. Enabling PersistentVolumes syncing is not necessary in order to use PersistentVolumeClaims.

Enable it if you have a scenario that require details about the PersistentVolumes and its IDs such when using Valero or specific CSI drivers.

See [Storage](/architecture/storage.mdx) for details on how this feature works.

### Sync all PersistentVolumes from virtual to host cluster
```YAML
sync:
  toHost:
    PersistentVolumes:
      enabled: true
```
